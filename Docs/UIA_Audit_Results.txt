UIA Accessibility Audit — VoiceBook Studio
==========================================
Audit completed: 2026-06-08
Auditor: Claude Code (claude-sonnet-4-6)
Build result after all changes: 0 errors, 0 warnings

NOTE: AutomationProperties.AccessibilityView is NOT a valid XAML attached
property in .NET 8 WPF. The correct approach for decorative elements is to
omit AutomationProperties.Name entirely — unnamed elements are skipped by
JAWS during Tab navigation and are not announced as named controls. This
approach was applied throughout.

RULE SUMMARY
------------
1. AutomationProperties.Name: meaningful, sentence-case, no type words in Name
2. AutomationProperties.HelpText: full sentence + Dragon voice command on all buttons
3. Every ListBox/ListView: has AutomationProperties.Name
4. Panel region Grids: have AutomationProperties.Name as landmark
5. Decorative containers/TextBlocks: Name omitted (JAWS skips unnamed elements)
6. Status TextBlocks with dynamic content: AutomationProperties.LiveSetting=Polite/Assertive
7. Non-interactive elements: IsTabStop=False where needed
8. Tab order: strictly sequential left panel → middle → right panel
9. Dialogs: AutomationProperties.Name on Window root; AnnouncementRegion live-region
   TextBlock added; code-behind must set AnnouncementRegion.Text in Window_Loaded

Pending code-behind work (not blocking build — functional gap for JAWS)
=============================================================================

Each dialog with an AnnouncementRegion TextBlock requires a code-behind change.
In Window_Loaded (or Loaded event handler), set:

    AnnouncementRegion.Text = "[dialog name] is open.";

This fires the UIA LiveRegion notification that makes JAWS speak the dialog name
immediately when focus enters the window — identical to how TutorialDialog works.

  Views/ApiKeyDialog.xaml.cs          → "Configure Anthropic API key dialog is open."
