# VoiceBook Studio — Dragon Professional MyCommands Setup Guide

This guide tells you exactly how to create every Dragon command needed to run
VoiceBook Studio entirely by voice. After completing it, you will be able to
do everything — navigate panels, manage chapters, write, save, get AI feedback,
export your book, and close the app — without touching the keyboard or mouse.

---

## How the Commands Work

VoiceBook has two types of Dragon commands:

**Type A — Direct Keystroke commands**
Dragon sends a keyboard shortcut directly to the app. Instant. One step.
Used for: save, panels, add chapter, feedback, new project, etc.

**Type B — Step-by-Step commands**
Dragon presses **Ctrl+Shift+Space** (which opens and clears the VoiceBook
command bar), types the command text, then presses Enter. The app reads the
text and executes the command.
Used for: pacing, dialogue, next chapter, insert at cursor, export, etc.

All commands are scoped to VoiceBook Studio only — they will not fire in any
other application.

---

## Before You Start — One-Time Setup

**Open Dragon Command Browser:**
1. Say "Open Command Browser" — or click Tools → Command Browser
2. Click **New** (or say "Click New")
3. You are now in the MyCommand editor

**For every command in this guide, you will:**
1. Click **New** in Command Browser
2. Set the **Name** field to exactly what it says in the "Name" column
3. Set the **Application** to: `VoiceBook Studio` (select from list, or type it)
   — This scopes the command so it only fires in VoiceBook
4. Set the **Command Type** as shown (Keystroke or Step-by-Step)
5. Fill in the action as described
6. Click **Save** (or say "Click Save")
7. Repeat for the next command

If VoiceBook is not in the Application list yet, launch it first, then come
back to the Command Browser and it will appear.

## Quick Reference — Most Used Commands After Setup

Once everything is created, your dad's daily workflow looks like this:

**Starting a writing session:**
- "Open project" or "Save VoiceBook" (if already open)
- "Go to editor" — start dictating normally

**While writing:**
- Just dictate — Dragon handles all text input natively
- "Save VoiceBook" every few minutes
- "Next chapter" / "Previous chapter" to move between chapters

**Getting feedback:**
- "Run feedback" — comprehensive feedback on the current chapter
- "Pacing feedback" / "Dialogue feedback" / "Style feedback"
- "Book analysis" — whole manuscript at once

**After Claude responds:**
- "Insert at cursor" / "Insert at end" — add to chapter
- "Save response card" — keep for later

**When done:**
- "Save VoiceBook"
- "Close VoiceBook"

## Troubleshooting

**Command fires in other apps:**
Make sure the Application field is set to VoiceBook Studio for every command.
If you set Application to "Global" by mistake, the command will fire everywhere.

**Step-by-Step command types the command text instead of running it:**
The command bar (Ctrl+Shift+Space) did not open in time. Increase the Wait
step from 300ms to 500ms or 700ms.

**"VoiceBook Studio" does not appear in the Application list:**
Launch VoiceBook Studio first, then go back to Command Browser and click the
Application dropdown — it should now be listed.

**Command is not recognised when you say it:**
Open Command Browser, find the command, and use Dragon's Train feature to
train it to your dad's voice on that phrase.

**Dragon types the phrase instead of running the command:**
The command is probably set to Global scope instead of VoiceBook Studio, or
the command name contains a word Dragon is treating as dictation.
Consider adding "VoiceBook" as a prefix to commands that conflict (e.g.
change "Save" to "Save VoiceBook").
