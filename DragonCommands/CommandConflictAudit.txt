METHODOLOGY
════════════════════════════════════════════════════════════════════════════════

Dragon MyCommands scoped to AppTitle "VoiceBook Studio" (startswith match)
override Dragon global built-in commands when VoiceBook Studio is the
foreground window.  Dragon priority order: App-specific > User global >
Dragon built-in global.

J-Say 24 commands are JAWS-side voice commands.  They are active only when
J-Say is enabled AND JAWS is running.  J-Say commands use a "j-say" spoken
prefix (e.g. "j-say help") and therefore do not conflict with Dragon commands
that lack that prefix.

CONFLICT RATINGS USED IN THIS TABLE
    NONE     — No conflict identified.
    LOW      — Phrase overlaps in some Dragon modes but app-scope takes
               priority or the phrase is contextually distinct.
    MODERATE — Dragon or J-Say has a near-identical built-in; the app-scope
               command will win but users should be aware of the override.
    HIGH     — Phrase is reserved in Dragon dictation mode; must not be used
               as a stand-alone voice command while the editor has focus.

SUMMARY — COMMANDS REQUIRING USER AWARENESS
════════════════════════════════════════════════════════════════════════════════

1.  "stop" (bare)
    Never registered in VoiceBookStudio_Commands.xml.
    Dragon uses "stop" to pause the microphone.  J-Say may intercept it.
    All stop-reading voice commands use "stop reading", "quiet", or "silence".
    ACTION: No XML change needed.  Document in user guide.

2.  "send"
    App-scoped command wins over Dragon's contextual "send" (email apps only).
    No action needed.  Low risk in this application context.

3.  "close application" vs Dragon "close window"
    These are different phrases.  "Close window" still routes to Dragon and
    triggers the standard Win32 close-window menu action.  "Close application"
    routes to VoiceBook Studio's graceful-close handler.
    ACTION: User guide should note both phrases and their different effects.

4.  "new chapter" vs Dragon "new line" / "new paragraph"
    These are phonetically distinct.  "New line" and "new paragraph" remain
    available for dictation inside the editor without conflict.
    No action needed.

5.  "read paragraph" vs JAWS Insert+P
    JAWS Insert+P reads the focused paragraph using screen-reader APIs.
    The Dragon voice command "read paragraph" routes to the application's
    paragraph-at-cursor reader (SAPI TTS).  These are independent mechanisms.
    When JAWS is detected, AudioFeedbackService.Speak() is suppressed so JAWS
    handles all audio — but the voice command still sets the status live region
    which JAWS announces.
    ACTION: When JAWS is running, advise user to prefer Insert+P over the
    Dragon voice command to avoid double-reading.

JAWS F6/F7 CONFLICT NOTE:
JAWS uses F6 (Next Heading) and F7 (Previous Heading) for document navigation.
These bindings are active when JAWS virtual-cursor mode is on (browse mode in
web browsers, etc.).  In VoiceBook Studio, JAWS should be in application/forms
mode because the window contains interactive controls.  In forms/application
mode, JAWS passes F6 and F7 through to the application rather than consuming
them for heading navigation.
VERIFICATION: Confirm with Kelly that F6/F7 chapter navigation works correctly
under JAWS in application mode.  If JAWS intercepts them, remap to Ctrl+F6 and
Ctrl+F7 in both MainWindow.xaml.cs and the Dragon XML script keystrokes.
